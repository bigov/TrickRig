Клиент-серверная часть на базе enet
====================================

I. Серверная часть.
-------------------

1. Всегда работа только в (фоновом) режиме сервиса (демона). Передача данных
производится через API локального/сетевого клинета и через SIG-налы управления.
Поддерживается режим отладки - когда все сообщения записываются в файл журнала.


I. Клиентская часть
-------------------

всегда используется GUI или консоль. Если доступно графическое окружение, то
запускается GIU интерфейс на базе FLTK, иначе - текстовая консоль.

1. Консольный режим.

Для ввода команды надо нажать [Enter]. При этом вывод данных от управляемого
сервера перенаправляется в файл/буфер (на время таймаута). Вывод данных на
экран восстанавливается после приема команды по клавише [Enter] или по
истечении таймаута.
