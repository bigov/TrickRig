Клиент-серверная часть на базе enet
====================================

I. Серверная часть.
-------------------

1. Всегда работа только в (фоновом) режиме сервиса (демона). Передача данных
производится только через API локального/сетевого клиента. Поведение зависит
от наличия активных сессий. Если сессий нет - производится автоматическое
завершение работы. Поддерживается режим отладки - когда все сообщения
записываются в файл журнала. Интерфейсов взаимодействия с оператором не
предусмотрено.


I. Клиентская часть
-------------------

Клиентская часть может быть автономным модулем (только для управления работой
серверной частью) или работать в составе основного приложения. Всегда
используется GUI или консоль. Если доступно графическое окружение, то
запускается GIU интерфейс на базе FLTK, иначе - текстовая консоль.

Можно использовать перенаправление потоков ввода/вывода на файлы.

1. Консольный режим.

Для ввода команды надо нажать [Enter]. При этом вывод данных от управляемого
сервера перенаправляется в файл/буфер (на время таймаута). Вывод данных на
экран восстанавливается после приема команды по клавише [Enter] или по
истечении таймаута.

2. GIU режим.

Интерфейс работает на базе FLTK - отдельно строка ввода команд управления,
отдельно окно вывода информации и сообщений.
