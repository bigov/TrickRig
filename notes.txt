
### ПРОБЛЕМА ПЛАВНОСТИ УПРАВЛЕНИЯ ПЕРЕМЕЩЕНИЕМ ###

Расчет положения камеры при перемещении:

- скорость перемещения Vn умножаем на время между кадрами dT

  Px = Vx * dT

Получаем при большом FPS малое время между кадрами - меньшее расстояние
перемещения. При уменьшении FPS соответственно увеличивается время между
кдрами - расстояние между расчетными точками положения увеличивается. Получаем
плавное перемещение камеры независимо от FPS. Данный алгоритм работает при
постоянной скорости движения (клавиша движения нажата или нет).

Покадровый расчет угла поворота камеры, управляемой курсором мыши, независимо
от колебаний FPS, должен учитывать не только длину смещения мыши, но и время
между кадрами, за которое это перемещение произошло. Получается, что нужно
учитывать скорость перемещения указателя.

### создание таблиц в базе данных для хранения информации о воксах ###

## В таблицах должны хранится значения

# Таблица размещения #

- координата: x, y, z (integer)
- индекс типа: - integer
- метка времени

# Таблица типов #

- размера вокса (ineger)
- цвет: R, G, B, A, 
- текстура
- тип вокса

## Пример команд SQLite для создания таблиц ##

CREATE TABLE "voxels" (
  "x"	INTEGER NOT NULL,
  "y"	INTEGER NOT NULL,
  "z"	INTEGER NOT NULL,
  "size"	INTEGER NOT NULL,
  "color"	INTEGER,
  "tехture" INTEGER
);
CREATE UNIQUE INDEX "point" ON "voxels" (
  "x"	ASC,
  "y"	ASC,
  "z"	ASC,
  "size" ASC
);
INSERT OR REPLACE INTO "voxels" ("x", "y", "z", "size") VALUES (0, 0, 0, 32);
